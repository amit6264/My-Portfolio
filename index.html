AWS Full-Stack End-to-End Infrastructure

PROJECT 02
🚀
End-to-End CI/CD Automation on EKS
Deployed on EKS

Built a fully automated end-to-end CI/CD pipeline for a containerized microservice application. Code changes trigger a Jenkins pipeline that builds a Docker image, runs unit tests, tags and pushes the image to Amazon ECR, updates the Helm chart values, and commits back to Git — where ArgoCD detects the change and automatically syncs and deploys the new image to Amazon EKS. Zero-downtime rolling updates with health checks and automatic rollback on failure.

Pipeline Flow
Code Push→ Jenkins Build→ Docker Build→ Push to ECR→ Update Helm→ ArgoCD Sync→ EKS Deploy
Jenkins Docker Amazon ECR ArgoCD Amazon EKS Helm Kubernetes GitOps GitHub
PROJECT 03
⚡
Jenkins + Docker + ECR + ArgoCD GitOps Deployment
GitOps Production

Implemented a GitOps-based deployment system using Jenkins as the CI engine and ArgoCD as the CD operator. Jenkins watches the application repository, runs the build pipeline, creates a versioned Docker image and pushes it to Amazon ECR. A separate Git config repo holds Kubernetes manifests — ArgoCD continuously monitors this repo and reconciles the live cluster state on EKS with the desired state in Git. Includes Slack notifications, environment-specific configs (dev/staging/prod), and automated rollback strategies.

GitOps Flow
App Repo→ Jenkins CI→ Docker Image→ ECR Push→ Config Repo→ ArgoCD CD→ K8s Cluster
Jenkins Docker Amazon ECR ArgoCD Kubernetes EKS GitOps Slack Alerts Multi-env
